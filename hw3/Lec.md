# Конспект лекции 3: Синдромное декодирование и границы кодов

## 1. Напоминание: основные обозначения

Рассматриваем линейный (n, k)-код над GF(2) (двоичный симметричный канал, ДСК).

Основные параметры:

    Скорость кода:       R = k/n
    Порождающая матрица: G, размер k x n
    Проверочная матрица: H, размер (n-k) x n = r x n, где r = n - k
    Соотношение:         G * H^T = 0   (нулевая матрица k x r)

Модель канала (ДСК): каждый бит инвертируется с вероятностью p, остается без изменения с вероятностью 1 - p.

---

## 2. Задача декодера

### Модель передачи

    Кодер: m -> c = m * G   (информационное слово -> кодовое слово)
    Канал: c -> y = c + e    (кодовое слово + вектор ошибки)
    Декодер: y -> c'         (оценка кодового слова)

Обозначения:

    c -- кодовое слово (вектор длины n, принадлежит коду)
    y -- принятая последовательность (вектор длины n)
    e -- вектор ошибки (вектор длины n), y = c + e

Задача декодера: найти кодовое слово, отличающееся от принятой двоичной последовательности y в наименьшем числе позиций (декодирование по ближайшему соседу).

---

## 3. Число исправляемых комбинаций ошибок

### Разбиение пространства на смежные классы

Вектор ошибки e -- любой двоичный вектор длины n. Всего таких векторов 2^n.

Кодовых слов в (n, k)-коде: 2^k.

Ключевое наблюдение: если взять фиксированный вектор e и прибавить его ко всем кодовым словам, получим множество из 2^k различных элементов:

    {c1 + e, c2 + e, ..., c_(2^k) + e}

Все элементы в этом множестве различны (иначе ci + e = cj + e => ci = cj).

### Смежные классы (классы эквивалентности)

При переборе всех возможных e мы получаем наборы по 2^k элементов. Эти наборы:

- Либо полностью совпадают, либо не пересекаются вовсе (из линейности кода).
- Каждый набор -- смежный класс (coset) кода.

Если e -- кодовое слово, то полученный набор = сам код (просто перенумерованный).

Число различных (непересекающихся) смежных классов:

    2^n / 2^k = 2^(n-k) = 2^r

Таким образом, все пространство GF(2)^n разбивается на 2^r смежных классов, в каждом по 2^k элементов.

### Почему классы не пересекаются

Код -- линейное подпространство. Множество {c + e : c принадлежит C} -- это сдвиг подпространства на вектор e. Два сдвига линейного подпространства либо совпадают, либо не пересекаются (аналог классов эквивалентности в алгебре).

Если два класса имеют хотя бы один общий элемент, то ci + e1 = cj + e2, откуда e1 - e2 = cj - ci -- кодовое слово, и оба класса совпадают целиком.

## 4. Синдромное декодирование

### Определение синдрома

Синдром принятого слова y:

    s = y * H^T

Размер синдрома: вектор длины r = n - k.

Свойства:

- Если y -- кодовое слово, то s = 0.
- s = y * H^T = (c + e) * H^T = c * H^T + e * H^T = 0 + e * H^T = e * H^T.

Таким образом, синдром зависит только от вектора ошибки e, а не от переданного кодового слова c.

Все элементы одного смежного класса имеют одинаковый синдром.

### Лидер смежного класса

Из каждого смежного класса выбираем представителя с минимальным весом -- лидер класса (coset leader). Это наиболее вероятный вектор ошибки для данного синдрома.

### Алгоритм синдромного декодирования

1. Предварительный этап (таблица синдромов):
   - Для каждого из 2^r возможных синдромов найти лидера смежного класса (вектор минимального веса с данным синдромом).
   - Записать таблицу: синдром -> лидер класса (предполагаемый вектор ошибки).

2. Декодирование:
   - Принять слово y из канала.
   - Вычислить синдром: s = y * H^T.
   - По таблице найти вектор ошибки e, соответствующий синдрому s.
   - Восстановить кодовое слово: c = y + e (сложение в GF(2) = XOR).

### Пример: код с проверкой на четность (3, 2)

Код: все слова четного веса длины 3.

    H = (1 1 1)   (матрица 1 x 3)

Кодовые слова: {000, 011, 101, 110}.

Порождающая матрица:

|   | 1 | 2 | 3 |
|---|---|---|---|
| g1| 1 | 0 | 1 |
| g2| 0 | 1 | 1 |

Все 8 двоичных векторов длины 3 и их синдромы s = y * H^T:

| y   | s |
|-----|---|
| 000 | 0 |
| 001 | 1 |
| 010 | 1 |
| 011 | 0 |
| 100 | 1 |
| 101 | 0 |
| 110 | 0 |
| 111 | 1 |

Смежные классы:

    s = 0: {000, 011, 101, 110} -- сам код
    s = 1: {001, 010, 100, 111}

Лидер класса s = 0: вектор 000 (вес 0).
Лидер класса s = 1: вектор 001, 010 или 100 (все веса 1) -- выбираем любой, например 001.

Но внимание: d_min = 2, поэтому код может только обнаружить одну ошибку, но не исправить. При синдроме s = 1 мы знаем, что ошибка есть, но не знаем где именно.

### Пример: код (3, 2) с проверкой на четность -- декодирование

Передаем кодовое слово c = 110. Ошибка e = 001. Принято y = 110 XOR 001 = 111.

    s = y * H^T = 111 * (1,1,1)^T = 1 + 1 + 1 = 1

Синдром = 1. По таблице лидер = 001.

    c' = y + e_лидер = 111 + 001 = 110

Кодовое слово восстановлено верно.

## 5. Радиус покрытия

### Определение

Радиус покрытия кода -- наибольшее число ошибок, которое код может исправить (для наихудшего случая среди всех исправляемых комбинаций).

Эквивалентно: радиус покрытия -- это наибольший вес среди всех лидеров смежных классов.

Для линейного кода достаточно рассматривать расстояния от нулевого кодового слова (из-за линейности картина симметрична для всех кодовых слов).

### Связь с d_min

Минимальное расстояние d_min определяет число гарантированно исправляемых ошибок:

    t = floor((d_min - 1) / 2)

Геометрическая интерпретация: вокруг каждого кодового слова -- сфера радиуса t. Условие d_min >= 2t + 1 гарантирует, что сферы не пересекаются: все, что попало в сферу, декодируется в центр этой сферы.

## 6. Задача упаковки пространства

Построение кода с заданной кратностью исправляемых ошибок t -- это задача упаковки пространства шарами радиуса t с центрами в кодовых словах.

Аналогия: как упаковать апельсины в ящик, чтобы поместилось максимальное количество.

Плотная сферическая упаковка -- когда вне сфер слов не остается, т.е. каждый вектор пространства попадает в некоторую сферу. Коды, достигающие такой упаковки, называются совершенными.

Нетривиальные совершенные коды:

- Коды Хэмминга
- Код Голея (23, 12)

---

## 7. Граница Хэмминга (граница сферической упаковки)

### Формулировка

Для любого q-ичного кода длины n с минимальным расстоянием d >= 2t + 1, число кодовых слов M удовлетворяет:

    M <= q^n / SUM(i=0..t) C(n,i) * (q-1)^i

Для двоичного кода (q = 2):

    M <= 2^n / SUM(i=0..t) C(n,i)

### Доказательство

Шар радиуса t с центром в точке x -- множество всех точек на расстоянии не более t от x.

Число точек в шаре радиуса t в пространстве GF(q)^n:

    V(n, t) = SUM(i=0..t) C(n,i) * (q-1)^i

Объяснение: на расстоянии i от центра находятся C(n,i) * (q-1)^i точек, потому что нужно выбрать i позиций из n (число сочетаний C(n,i)), и в каждой позиции значение может измениться на одно из (q-1) значений.

Для двоичного случая (q = 2):

    V(n, t) = SUM(i=0..t) C(n,i)

Так как d >= 2t + 1, сферы радиуса t вокруг различных кодовых слов не пересекаются. Суммарный объем всех сфер не превышает общее число точек:

    M * V(n, t) <= q^n

Отсюда:

    M <= q^n / V(n, t)

QED

### Совершенные коды

Равенство в границе Хэмминга достигается (для нетривиальных линейных кодов) только для:

- Кодов Хэмминга: n = 2^r - 1, k = 2^r - 1 - r, d = 3, t = 1
- Кода Голея: (23, 12)-код с d = 7, t = 3

Для них M * V(n, t) = q^n -- все точки пространства покрыты сферами, код совершенный.

## 8. Граница Варшамова-Гилберта

### Формулировка

Если выполняется неравенство:

    q^(n-k) > SUM(i=0..d-2) C(n-1, i) * (q-1)^i

то существует q-ичный (n, k)-код с минимальным расстоянием не менее d.

Для двоичного кода (q = 2):

    2^(n-k) > SUM(i=0..d-2) C(n-1, i)

### Идея доказательства (конструктивное построение)

Строим проверочную матрицу H (размер r x n, где r = n - k) столбец за столбцом:

1. Первый столбец: берем любой ненулевой вектор длины r.

2. Второй столбец: берем любой вектор, не кратный первому.

3. Третий столбец: берем вектор, не являющийся линейной комбинацией первых двух.

4. На шаге i: берем вектор, не являющийся линейной комбинацией никаких (d-2) или менее из уже выбранных (i-1) столбцов.

Это гарантирует, что любые (d-1) столбцов H линейно независимы, а значит d_min >= d.

### Подсчет исключаемых векторов

На шаге добавления i-го столбца нужно исключить:

- Нулевой вектор: 1 штука
- Кратные одному из (i-1) столбцов: (i-1) * (q-1)
- Линейные комбинации пар: C(i-1, 2) * (q-1)^2
- ...
- Линейные комбинации (d-2) столбцов: C(i-1, d-2) * (q-1)^(d-2)

Всего исключается не более SUM(j=0..d-2) C(i-1, j) * (q-1)^j векторов.

Если q^r (общее число векторов длины r) больше числа исключаемых, то подходящий столбец всегда можно найти.

Подставляя максимальное i = n (последний столбец), получаем условие из формулировки теоремы.

### Смысл границы

Граница Варшамова-Гилберта -- граница существования: она гарантирует, что код с заданными параметрами существует. Это нижняя граница на число кодовых слов (или эквивалентно, нижняя граница на размерность k при заданных n и d).

В отличие от границы Хэмминга (верхняя граница: код не может быть лучше), граница Варшамова-Гилберта -- нижняя граница (код не хуже этого точно существует).
