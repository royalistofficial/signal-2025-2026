# Конспект лекции 5: Построение решётки кода по порождающей матрице

## 1. Завершение примера: решётка кода (6, 3)

### Напоминание

Код (6, 3) с 8 кодовыми словами длины 6. Решётка строится ярус за ярусом: на каждом ярусе выписываем все варианты "прошлого" (первые i символов кодовых слов) и соответствующие множества "будущего".

### Построение ярусов 0-3 (повторение)

    Ярус 0: 1 узел, будущее = все 8 кодовых слов.
    Ярус 1: 2 узла (первый символ = 0 или 1), по 4 будущих слова в каждом.
    Ярус 2: 4 узла (первые два символа: 00, 01, 10, 11), по 2 будущих.
    Ярус 3: 4 узла (первые три символа: 000, 011, 101, 110), по 2 будущих.

### Ярус 4: первое схлопывание

На четвёртом ярусе рассматриваем первые 4 символа. У нас 8 четвёрок (по числу кодовых слов), но некоторые имеют одинаковое будущее (одинаковые два последних символа).

Например, кодовые слова с прошлым 0000 и 1101 имеют общее будущее {00}. Поэтому их объединяем в один узел.

Аналогично объединяются другие пары. В итоге на ярусе 4 получаем 4 узла (вместо возможных 8).

### Ярус 5: дальнейшее схлопывание

На пятом ярусе будущее -- всего один символ (0 или 1). Получаем только 2 узла.

### Ярус 6: завершение

Один узел (конечная точка). Будущее пусто.

### Итоговый профиль решётки

    Профиль: (1, 2, 4, 4, 4, 2, 1)

Каждый путь от начала до конца решётки соответствует кодовому слову.

## 2. Декодирование по решётке (пример)

### Постановка

Код (6, 3). Из канала принято y = (1, 0, 1, 0, 0, 1).

### Алгоритм

На каждом ребре вычисляем расстояние Хэмминга между принятым символом и меткой ребра. Накапливаем суммарный вес на каждом узле.

Ярус 0 -> 1: y1 = 1.

    Ребро "0": d(1, 0) = 1. Вес = 1.
    Ребро "1": d(1, 1) = 0. Вес = 0.

Ярус 1 -> 2: y2 = 0. Вычисляем для каждого ребра, суммируем с накопленным весом.

Ярус 2 -> 3: y3 = 1. Аналогично.

Ярус 3 -> 4: y4 = 0. На этом ярусе происходит слияние путей. Из двух входящих путей в каждый узел оставляем путь с меньшим весом (отбрасываем худший).

Ярус 4 -> 5: y5 = 0. Снова слияние, отбрасывание проигрышных путей.

Ярус 5 -> 6: y6 = 1. Финальный выбор.

### Результат

Декодированное кодовое слово: c = (1, 0, 1, 1, 0, 1) с расстоянием Хэмминга 1 от принятого y = (1, 0, 1, 0, 0, 1).

Проверка: ошибка произошла только в 4-й позиции (0 вместо 1).

## 3. Профиль сложности решётки

### Определение

Профиль сложности решётки -- вектор, указывающий число узлов на каждом ярусе.

Для нашего кода (6, 3): профиль = (1, 2, 4, 4, 4, 2, 1).

### Минимальная решётка

Признак минимальности: на каждом ярусе все пути, имеющие общее будущее (или общее прошлое), проходят через один и тот же узел.

Всякий код имеет минимальную решётку (с точностью до нумерации узлов).

Важно: алгоритм построения из предыдущей лекции (перебор всех кодовых слов) работает для любого кода (не обязательно линейного), но имеет экспоненциальную сложность по числу кодовых слов.

## 4. Построение решётки по порождающей матрице (для линейных кодов)

### Мотивация

Для линейных кодов можно построить решётку гораздо эффективнее -- непосредственно из порождающей матрицы G, без перечисления всех кодовых слов.

### Определения

Для последовательности x = (x1, x2, ..., xn):

    begin(x) -- номер позиции первого ненулевого элемента (начало)
    end(x) -- номер позиции последнего ненулевого элемента (конец)
    Активные элементы -- элементы на позициях от begin(x) до end(x)
    span(x) -- число активных элементов = end(x) - begin(x) + 1

Пример: x = (0, 0, 1, 0, 1, 1, 0, 0).

    begin(x) = 3
    end(x) = 6
    Активные позиции: 3, 4, 5, 6
    span(x) = 4

## 5. Минимальная спановая форма (МСФ)

### Определение

Порождающая матрица G находится в минимальной спановой форме (МСФ), если все начала (begin) и все концы (end) строк матрицы различны.

### Приведение к МСФ

Выполняем элементарные строчные преобразования (сложение строк, перестановка строк), добиваясь:

1. Сначала -- различных начал: уничтожаем крайние левые единицы, складывая строки.
2. Затем -- различных концов: уничтожаем крайние правые единицы.

### Пример

Исходная матрица G:

|   | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|
| g1| 1 | 0 | 1 | 1 | 0 | 1 |
| g2| 0 | 1 | 1 | 0 | 1 | 0 |
| g3| 1 | 0 | 1 | 0 | 0 | 1 |

Шаг 1. g3 := g3 + g1 = (0, 0, 0, 1, 0, 0). Теперь начала: g1 на поз. 1, g2 на поз. 2, g3 на поз. 4 -- все разные.

Переупорядочим для красоты (g2 и g3 местами):

    g1 = (1, 0, 1, 1, 0, 1),  g2 = (0, 0, 0, 1, 0, 0),  g3 = (0, 1, 1, 0, 1, 0)

Нет, лучше так: g1, g3 (бывшая g2), g2 (бывшая g3):

    g1 = (1, 0, 1, 1, 0, 1),  g2 = (0, 1, 1, 0, 1, 0),  g3 = (0, 0, 0, 1, 0, 0)

Начала: 1, 2, 4. Концы: 6, 5, 4 -- все разные? Нет, надо проверить.

Шаг 2. g1 := g1 + g2 = (1, 1, 0, 1, 1, 1) -- нет, лучше g1 := g1 + g3 для уничтожения совпадающих концов.

Конкретно для данного примера (результат после полного приведения):

    g1 = (1, 1, 0, 0, 0, 0)    begin=1, end=2
    g2 = (0, 1, 1, 1, 1, 0)    begin=2, end=5
    g3 = (0, 0, 0, 1, 1, 1)    begin=4, end=6

Все начала различны (1, 2, 4). Все концы различны (2, 5, 6). Это МСФ.

### Активные строки на каждой позиции

|Позиция|Активные строки|Информационные символы|
|-------|---------------|---------------------|
| 1     | g1            | m1                  |
| 2     | g1, g2        | m1, m2              |
| 3     | g2            | m2                  |
| 4     | g2, g3        | m2, m3              |
| 5     | g2, g3        | m2, m3              |
| 6     | g3            | m3                  |

## 6. Алгоритм построения решётки по МСФ

### Правило определения узлов

Узлы на ярусе i нумеруются значениями информационных символов, соответствующих строкам, активным в позиции i.

Число узлов на ярусе i = 2^(число активных строк в позиции i).

### Правило определения рёбер

Кодовый символ на ребре (переход из узла в узел) вычисляется как линейная комбинация активных элементов соответствующего столбца порождающей матрицы с коэффициентами из информационных символов.

### Построение решётки для примера

Из МСФ:

    g1 = (1, 1, 0, 0, 0, 0)
    g2 = (0, 1, 1, 1, 1, 0)
    g3 = (0, 0, 0, 1, 1, 1)

Число узлов по ярусам:

    Ярус 0: 1 (нет активных строк)
    Ярус 1: 2 (активна g1: m1 = 0 или 1)
    Ярус 2: 4 (активны g1, g2: m1m2 = 00, 01, 10, 11)
    Ярус 3: 4 (активны g1, g2: но g1 заканчивается -> g2: m2 = 0,1; и g2 -> но учёт: фактически 4)
    Ярус 4: 4 (активны g2, g3: m2m3 = 00, 01, 10, 11)
    Ярус 5: 2 (активна g3: m3 = 0 или 1)
    Ярус 6: 1 (нет активных строк)

Профиль: (1, 2, 4, 4, 4, 2, 1) -- совпадает с построенным ранее!

### Кодовые символы на рёбрах

Для каждого ребра: берём столбец i матрицы G, оставляем только элементы активных строк, умножаем на соответствующие информационные символы (mod 2).

Пример: ярус 1, позиция 1. Активна только g1. Столбец 1: g1[1] = 1.

    m1 = 0: кодовый символ = 0 * 1 = 0
    m1 = 1: кодовый символ = 1 * 1 = 1

Пример: ярус 2, позиция 2. Активны g1, g2. Столбец 2: g1[2] = 1, g2[2] = 1.

    m1=0, m2=0: символ = 0*1 + 0*1 = 0
    m1=0, m2=1: символ = 0*1 + 1*1 = 1
    m1=1, m2=0: символ = 1*1 + 0*1 = 1
    m1=1, m2=1: символ = 1*1 + 1*1 = 0

## 7. Секционная решётка

Если на нескольких соседних ярусах набор активных строк не меняется (никакая строка не начинается и не заканчивается), эти ярусы можно объединить в один "секционный" ярус.

На рёбрах секционной решётки лежит не один символ, а несколько (кортеж символов).

Для нашего примера: ярусы 4 и 5 можно объединить (на обоих активны g2, g3).

Профиль секционной решётки: (1, 2, 4, 4, 2, 1) -- на один ярус меньше.

## 8. Теорема о минимальности решётки из МСФ

### Формулировка

Решётка, построенная из порождающей матрицы в минимальной спановой форме, является минимальной.

### Доказательство (схема)

Нужно показать: пути с одинаковым будущим проходят через один узел.

Узел на ярусе L определяется значениями информационных символов, соответствующих активным на уровне L строкам.

Активные строки линейно независимы (по построению МСФ: все начала и концы различны, поэтому нельзя выразить одну строку через другие на данном промежутке).

Все активные строки заканчиваются на ярусах с номером > L. Значит, любая нетривиальная линейная комбинация этих строк отлична от нуля хотя бы на одной позиции > L.

Следовательно: если два кодовых слова проходят через разные узлы на ярусе L (разные значения активных информационных символов), то их будущие части обязательно различаются.

Контрапозитивно: одинаковое будущее => одинаковые активные информационные символы => один и тот же узел.   QED

## 9. Сравнение двух методов построения решётки

| Метод | Входные данные | Сложность | Применимость |
|-------|---------------|-----------|-------------|
| Перебор кодовых слов | Все M кодовых слов | O(M * n) = O(2^k * n) | Любой код |
| По порождающей матрице | G в МСФ (k строк) | O(k * n) | Только линейные коды |

Для линейных кодов метод через МСФ экспоненциально эффективнее.
