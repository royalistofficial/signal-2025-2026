# Конспект лекции 9: БЧХ-коды и исправление двух ошибок

## 1. Разбор ДЗ: три способа построения решётки

Напоминание: одну и ту же решётку линейного кода можно построить тремя способами.

1. **По кодовым словам.** Линейность кода не используется -- перечисляются все кодовые слова, на каждом ярусе формируются узлы по совпадающему "будущему". Нумерация узлов произвольная.

2. **По порождающей матрице G.** Матрица приводится к минимальной спановой форме (МСФ) перестановками и сложением строк. Узлы на ярусах нумеруются по правилу (значениями активных информационных символов), а не произвольно.

3. **По проверочной матрице H (синдромная решётка).** Узлы нумеруются накопленными синдромами.

Все три способа дают одну и ту же решётку с точностью до перенумерации узлов и расположения на картинке. Минимальную решётку можно получать любым из этих способов; у некоторых кодов её специальной "ухищрённой" перестановкой можно ещё дополнительно минимизировать, но в целом структура одна.

## 2. Разбор ДЗ: порядок элементов в GF(5^2) и кодер на сдвиговом регистре

### Порядки элементов

По определению порядок элемента -- наименьшая степень, в которой нужно его возвести, чтобы получить единицу.

- Если k взаимно просто с 24, то порядок X^k равен 24 (наименьшее такое значение).
- В общем случае порядок X^k равен 24 / gcd(k, 24).

Получается, что в группе порядка 24 порядки элементов:

- 8, 16 -- порядок 3
- 6, 18 -- порядок 4
- 4, 20 -- порядок 6
- 3, 9, 15, 21 -- порядок 8
- 2, 10, 14, 22 -- порядок 12

(И так далее по таблице.)

### Кодер на сдвиговом регистре

Дано продолжение моделирования регистра с h(x) = 1 + x + x^4 от состояния (0, 0, 1, 1). Требовалось:

- доделать смену состояний;

## 3. Введение: БЧХ-коды и коды Рида-Соломона

Начинаем большую тему -- БЧХ-коды (Боуза-Чоудхури-Хоквингема) и коды Рида-Соломона (РС).

- БЧХ-коды -- наиболее часто используемые в системах хранения и передачи данных.
- Известен хороший способ построения и хороший способ декодирования с полиномиальной сложностью.
- Канал, в котором мы работаем, -- двоичный (на входе и выходе нули и единицы).

Коды Рида-Соломона:

- Имеют максимально возможное расстояние, удовлетворяющее границе Синглтона: d <= n - k + 1, причём с равенством.
- Это значит, что они оптимальны по этому параметру.
- Их минус: алфавит кода растёт с длиной кода. Это уже не двоичные коды.

Мы сейчас идём от БЧХ-кодов, потом перейдём к Рида-Соломона.

## 5. Напоминание: код Хэмминга через примитивный полином

### Двоичная проверочная матрица

У нас был код длины n = 7 (код (7, 4)). Проверочная матрица H -- 3 столбца x 7 -- содержала все ненулевые двоичные векторы длины 3, расположенные так, чтобы синдром при одиночной ошибке давал номер ошибочной позиции.

### Полиномиальное представление столбцов

Каждый столбец H -- это коэффициенты некоторого полинома над GF(2). Например (для одной из нумераций):

- 1 -> 1
- x -> 1 + x  (нет, см. ниже -- порядок зависит от выбора примитивного полинома)
- 1 + x
- x^2
- x^2 + 1
- x^2 + x
- x^2 + x + 1

Все 7 ненулевых полиномов степени < 3.

### Расширение поля и примитивный элемент

Расширяем GF(2) до GF(2^3) с помощью примитивного полинома p(x) = 1 + x + x^3. Этот полином неприводим над GF(2) (подстановка 0 даёт 1, подстановка 1 даёт 1 -- корней нет), приведённый (старший коэффициент 1).

В расширенном поле GF(2^3) есть примитивный элемент alpha, степени которого пробегают всю мультипликативную группу:

| Степень alpha | Полином     |
|---------------|-------------|
| alpha^0       | 1           |
| alpha^1       | x           |
| alpha^2       | x^2         |
| alpha^3       | 1 + x       |
| alpha^4       | x + x^2     |
| alpha^5       | 1 + x + x^2 |
| alpha^6       | 1 + x^2     |

(При alpha^3 получаем x^3, который заменяется на 1 + x по модулю p(x).)

### Проверочная матрица в виде степеней alpha

Проверочную матрицу H можно записать через степени примитивного элемента:

    H = (1, alpha, alpha^2, alpha^3, alpha^4, alpha^5, alpha^6)

Это та же матрица, только в другом виде: каждый элемент alpha^i при переходе к двоичному представлению превращается в столбец из 3 битов.

В двоичной форме (3 строки x 7 столбцов):

|     | alpha^0 | alpha^1 | alpha^2 | alpha^3 | alpha^4 | alpha^5 | alpha^6 |
|-----|---------|---------|---------|---------|---------|---------|---------|
| h1  | 1       | 0       | 0       | 1       | 0       | 1       | 1       |
| h2  | 0       | 1       | 0       | 1       | 1       | 1       | 0       |
| h3  | 0       | 0       | 1       | 0       | 1       | 1       | 1       |

## 7. Декодирование одной ошибки -- напоминание

Пусть c(x) -- кодовое слово, b(x) = c(x) + e(x) -- слово, пришедшее из канала с ошибкой. Декодер должен восстановить c(x).

Синдром принятого слова:

    s = b * H^T = SUM(i=0..n-1) b_i * alpha^i = b(alpha)

Поскольку c(x) -- кодовое слово, c(alpha) = 0, значит:

    s = c(alpha) + e(alpha) = 0 + e(alpha) = e(alpha)

Если ошибка в позиции i, то e(x) = x^i, и:

    s = alpha^i

Степень alpha в синдроме указывает позицию ошибки. Это и есть способ декодирования одиночных ошибок.

## 8. Случай двух ошибок

Пусть произошли две ошибки в разных позициях i и j (i != j). Тогда:

    e(x) = x^i + x^j

    s = alpha^i + alpha^j

Это одно уравнение с двумя неизвестными. У него огромное число решений -- одного синдрома недостаточно.

### Идея БЧХ: добавить ещё одну строку в H

Допишем к H ещё одну строку с какими-то элементами beta_0, beta_1, ..., beta_14:

    H = | 1       alpha   alpha^2  ...  alpha^14 |
        | beta_0  beta_1  beta_2   ...  beta_14  |

Тогда синдром распадается на две части:

    s_1 = alpha^i + alpha^j   (по верхней строке)
    s_2 = beta_i + beta_j     (по нижней строке)

Получаем систему из двух уравнений с двумя неизвестными. Если правильно выбрать beta-элементы (как функцию от alpha), систему можно решить.

В двоичной форме каждый beta_i -- это вектор из 4 битиков, и матрица H будет иметь 8 строк.

### Какую функцию beta_i = f(alpha^i) выбрать?

Не подходят:

- **f(z) = const * z** -- линейная зависимость, нулевой определитель в системе, ничего не получим.
- **f(z) = z^2** -- в поле характеристики 2 действует тождество (a + b)^2 = a^2 + b^2 (потому что 2ab = 0). Из-за этого второе уравнение оказывается просто квадратом первого и линейно зависимым: ничего нового не даёт.

Подходит **f(z) = z^3** -- кубическая функция. Возведение в куб не "линеаризуется" в GF(2).

Итак, нижняя строка H -- это:

    1, alpha^3, alpha^6, alpha^9, alpha^12, 1, alpha^3, alpha^6, alpha^9, alpha^12, 1, alpha^3, alpha^6, alpha^9, alpha^12

(Цикл 1, alpha^3, alpha^6, alpha^9, alpha^12 повторяется -- степени берутся по модулю 15.)

## 9. Решение системы для двух ошибок

Обозначим X = alpha^i, Y = alpha^j -- неизвестные степени, соответствующие позициям ошибок. Получаем систему:

    X + Y    = s_1
    X^3 + Y^3 = s_2

При одной или двух ошибках X != Y, поэтому s_1 != 0. Делим второе уравнение на первое:

    (X^3 + Y^3) / (X + Y) = s_2 / s_1

В GF(2) разложение:

    X^3 + Y^3 = (X + Y) * (X^2 + XY + Y^2)

(плюс перекрёстный член -X*Y*Y - Y*X*X пропадает в GF(2)). Сокращаем (X + Y):

    X^2 + XY + Y^2 = s_2 / s_1

В поле характеристики 2: (X + Y)^2 = X^2 + Y^2, значит X^2 + Y^2 = s_1^2. Подставляем:

    s_1^2 + XY = s_2 / s_1

    XY = s_2 / s_1 + s_1^2     (минус и плюс совпадают в GF(2))

Получили выражение для XY. Заодно X + Y = s_1 уже известно.

### Теорема Виета

Корни X, Y -- решения квадратного уравнения:

    z^2 + sigma_1 * z + sigma_2 = 0

где:

    sigma_1 = X + Y = s_1
    sigma_2 = X * Y = s_2 / s_1 + s_1^2

В конечном поле решать квадратное уравнение можно перебором: подставлять последовательно alpha^0, alpha^1, ..., alpha^14 и смотреть, при каких степенях получается 0. Степени корней дают позиции ошибок.

## 10. Конкретный пример декодирования

### Построение поля GF(2^4) через p(x) = 1 + x + x^4

В поле 16 элементов: 0 и 15 ненулевых.

| Степень  | Полином                  | Двоичная запись (b3 b2 b1 b0) |
|----------|--------------------------|-------------------------------|
| 0        | 0                        | 0 0 0 0                       |
| alpha^0  | 1                        | 0 0 0 1                       |
| alpha^1  | x                        | 0 0 1 0                       |
| alpha^2  | x^2                      | 0 1 0 0                       |
| alpha^3  | x^3                      | 1 0 0 0                       |
| alpha^4  | 1 + x                    | 0 0 1 1                       |
| alpha^5  | x + x^2                  | 0 1 1 0                       |
| alpha^6  | x^2 + x^3                | 1 1 0 0                       |
| alpha^7  | 1 + x + x^3              | 1 0 1 1                       |
| alpha^8  | 1 + x^2                  | 0 1 0 1                       |
| alpha^9  | x + x^3                  | 1 0 1 0                       |
| alpha^10 | 1 + x + x^2              | 0 1 1 1                       |
| alpha^11 | x + x^2 + x^3            | 1 1 1 0                       |
| alpha^12 | 1 + x + x^2 + x^3        | 1 1 1 1                       |
| alpha^13 | 1 + x^2 + x^3            | 1 1 0 1                       |
| alpha^14 | 1 + x^3                  | 1 0 0 1                       |

После alpha^14 цикл замыкается: alpha^15 = 1 = alpha^0.

### Передача и приём

Передано кодовое слово (15 битов; берём ненулевое для усложнения):

    c = 1 1 1 0 0 1 1 0 1 1 1 0 0 1 0

(Дано как кодовое; проверять не будем.)

В канале происходят две ошибки -- в позициях 2 и 9 (нумерация с 0). Принимаемое слово:

    b = 1 1 0 0 0 1 1 0 1 0 1 0 0 1 0
                ^               ^
             поз. 2          поз. 9
        (была 1, стала 0)  (была 1, стала 0)

(Точные биты в примере были: на позиции 2 пришёл 0, на позиции 9 пришёл 0.)

### Вычисление s_1

    s_1 = SUM(i: b_i = 1) alpha^i

Альтернативный подход: если бы все позиции с единицами кодового слова дали в сумме ноль, то отличие s_1 от нуля -- это вклад только тех позиций, где произошла ошибка:

    s_1 = alpha^2 + alpha^9

В двоичных полиномах:

    alpha^2 = x^2
    alpha^9 = x + x^3

    Сумма: x + x^2 + x^3 = alpha^11

Итак: s_1 = alpha^11.

### Вычисление s_2

Аналогично, по нижней строке H (где beta_i = alpha^{3i}):

    s_2 = (alpha^2)^3 + (alpha^9)^3 = alpha^6 + alpha^12

В двоичных полиномах:

    alpha^6  = x^2 + x^3
    alpha^12 = 1 + x + x^2 + x^3

    Сумма: 1 + x = alpha^4

Итак: s_2 = alpha^4.

### Вычисление sigma_1 и sigma_2

    sigma_1 = s_1 = alpha^11

    sigma_2 = s_2 / s_1 + s_1^2
            = alpha^4 / alpha^11 + (alpha^11)^2
            = alpha^(-7) + alpha^22
            = alpha^8 + alpha^7        (степени по модулю 15: -7 ≡ 8, 22 ≡ 7)

Складываем в полиномах:

    alpha^7 = 1 + x + x^3
    alpha^8 = 1 + x^2

    Сумма: x + x^2 + x^3 = alpha^11

Итак: sigma_2 = alpha^11.

### Квадратное уравнение

    z^2 + alpha^11 * z + alpha^11 = 0

### Проверка корней

Корни должны быть alpha^2 и alpha^9 (степени равны позициям ошибок).

**Проверка z = alpha^2:**

    (alpha^2)^2 + alpha^11 * alpha^2 + alpha^11
    = alpha^4 + alpha^13 + alpha^11

В полиномах:

    alpha^4  = 1 + x
    alpha^13 = 1 + x^2 + x^3
    alpha^11 = x + x^2 + x^3

    Сумма: (1+1) + (x+x) + (x^2+x^2) + (x^3+x^3) = 0   ok

**Проверка z = alpha^9:**

    (alpha^9)^2 + alpha^11 * alpha^9 + alpha^11
    = alpha^18 + alpha^20 + alpha^11
    = alpha^3 + alpha^5 + alpha^11

В полиномах:

    alpha^3  = x^3
    alpha^5  = x + x^2
    alpha^11 = x + x^2 + x^3

    Сумма: x^3 + (x+x^2) + (x+x^2+x^3) = (x+x) + (x^2+x^2) + (x^3+x^3) = 0   ok

Оба корня подтверждены. Остальные степени alpha при подстановке нуля не дают (на лекции это утверждается без проверки -- результат известен).

### Итог декодирования

Ошибки произошли в позициях 2 и 9 -- эти биты надо инвертировать в принятом слове, и получится исходное кодовое слово.
