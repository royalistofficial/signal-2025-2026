## 2. Введение: коды Рида-Соломона

Сегодня -- последняя теоретическая тема: **коды Рида-Соломона** (РС-коды).

### Где применяются

Эти коды используются повсеместно:

- считывание данных с жёстких дисков и компакт-дисков;
- передача данных в космической связи;
- мобильная связь.

### Главное отличие от рассмотренных ранее БЧХ-кодов

БЧХ-коды, которые мы рассматривали, были **двоичные**: даже когда мы расширяли GF(2) до GF(2^m) и работали со степенями alpha, мы потом разворачивали каждый элемент в столбец из нулей и единиц -- двоичный канал на входе и выходе.

Коды Рида-Соломона -- это **недвоичные** коды. На вход и выход подаются не нулики/единички, а элементы поля GF(q).

### Зачем недвоичный канал

1. Если физический канал использует не двоичную модуляцию (передаётся не "есть сигнал / нет сигнала", а один из q признаков), естественно использовать недвоичный код.

2. В рамках одного символа недвоичного алфавита может произойти сразу несколько двоичных ошибок. Если ошибки сгруппированы, недвоичный код, исправляющий один символ, фактически исправляет несколько подряд идущих двоичных ошибок. Для пакетных ошибок это особенно эффективно.

## 3. Минимизация избыточности через выбор поля

### Напоминание про БЧХ-коды

Порождающий полином БЧХ-кода:

    g(x) = НОК(M_b(x), M_{b+1}(x), ..., M_{b+d-2}(x))

Чем меньше степень каждого минимального многочлена, тем меньше степень g(x), тем меньше число проверочных символов n - k и тем меньше избыточность.

### Минимально возможная степень минимального многочлена

Минимальный многочлен элемента alpha над полем -- многочлен наименьшей степени с коэффициентами из этого поля, корнем которого является alpha.

Самая маленькая возможная степень -- 1: m(x) = x - alpha. Но коэффициенты должны лежать в исходном поле, значит сам элемент alpha тоже должен принадлежать исходному полю.

### Идея

Если работать сразу в большом поле GF(q) (а не строить расширение GF(p) -> GF(p^m)), то любой элемент alpha из GF(q) имеет минимальный многочлен степени 1. Все произведения вида (x - alpha^i) будут минимальными многочленами, и степень g(x) станет ровно d - 1.

На этом и основаны коды Рида-Соломона: мы не расширяем поле, а сразу работаем в нужном (недвоичном) поле GF(q).

## 4. Определение кода Рида-Соломона

Код Рида-Соломона -- код БЧХ над полем GF(q), у которого:

- алфавит кода -- само поле GF(q);
- длина n = q - 1 (максимально возможная при использовании степеней примитивного элемента в качестве позиций);
- порождающий полином имеет вид:

      g(x) = (x - alpha^b) * (x - alpha^(b+1)) * ... * (x - alpha^(b+d-2))

где alpha -- примитивный элемент GF(q), b -- начальная степень (часто b = 1).

### Почему длина n = q - 1

Позиции кодового слова соответствуют степеням переменной x. Чтобы все позиции различались, число позиций не должно превышать число различных ненулевых элементов поля, то есть q - 1.

---

## 5. Параметры РС-кода. Граница Синглтона

В порождающем полиноме d - 1 множителей вида (x - alpha^i), значит:

    deg g(x) = d - 1
    k = n - (d - 1) = n - d + 1

Откуда:

    d = n - k + 1

### Связь с границей Синглтона

Из лекции 2 известна **граница Синглтона**: для любого линейного (n, k)-кода

    d_min <= n - k + 1

У РС-кодов в этом неравенстве достигается **равенство**. Поэтому коды Рида-Соломона -- это коды с **максимально достижимым расстоянием** (MDS-коды). По параметру минимального расстояния они оптимальны.

## 7. Алгоритм декодирования Питерсона-Горенстейна-Цирлера

Этот алгоритм работает как для двоичных, так и для недвоичных БЧХ-кодов (включая РС). Будем рассматривать общий случай поля GF(q).

### Постановка

Кодовое слово в полиномиальной форме:

    c(x) = c_0 + c_1 * x + ... + c_{n-1} * x^(n-1)

(коэффициенты c_i -- элементы GF(q)).

В канале происходит ошибка:

    e(x) = e_0 + e_1 * x + ... + e_{n-1} * x^(n-1)

Из канала приходит:

    v(x) = c(x) + e(x) = v_0 + v_1 * x + ... + v_{n-1} * x^(n-1)

### Синдромы

По построению РС-кода: alpha, alpha^2, ..., alpha^(d-1) -- корни g(x), а значит и любого кодового слова. То есть c(alpha^j) = 0 для j = 1, ..., d-1.

Определяем синдромы:

    S_j = v(alpha^j) = c(alpha^j) + e(alpha^j) = 0 + e(alpha^j) = e(alpha^j)

То есть S_j зависит только от ошибки.

### Локаторы и значения ошибок

Пусть произошло nu ошибок (nu <= t = floor((d-1)/2)) на позициях i_1, i_2, ..., i_nu.

Введём обозначения:

    Y_h = e_{i_h}            -- значение ошибки на h-й позиции
    X_h = alpha^(i_h)        -- локатор h-й ошибки

Y_h -- сама величина искажения (в недвоичном случае это не просто XOR, а конкретный элемент поля). X_h -- степень alpha, по которой определяется позиция ошибки.

Тогда:

    S_j = Y_1 * X_1^j + Y_2 * X_2^j + ... + Y_nu * X_nu^j

для j = 1, 2, ..., d - 1. Получаем систему d - 1 уравнений с 2*nu неизвестными (Y и X). Система **нелинейная** относительно X_h (входят в степени), и сверх того само nu (число ошибок) заранее неизвестно.

---

## 8. Полином локатора ошибок

### Идея замены переменных

Вместо того чтобы искать сами X_h, ищем коэффициенты полинома, корнями которого являются обратные величины X_h^(-1).

Полином локатора ошибок:

    Lambda(x) = PROD(h=1..nu) (1 - x * X_h)

При подстановке x = X_h^(-1) одна из скобок обращается в ноль, значит X_h^(-1) -- корни Lambda(x).

Раскрывая скобки:

    Lambda(x) = 1 + lambda_1 * x + lambda_2 * x^2 + ... + lambda_nu * x^nu

Свободный член равен 1 (это особенность данной нормировки).

### Получение линейной системы

Подставим в Lambda(x) значение x = X_h^(-1) (для любого h = 1, ..., nu):

    1 + lambda_1 * X_h^(-1) + lambda_2 * X_h^(-2) + ... + lambda_nu * X_h^(-nu) = 0

Умножим обе части на X_h^(j+nu) * Y_h (где j -- параметр, который мы выберем):

    X_h^(j+nu) * Y_h + lambda_1 * X_h^(j+nu-1) * Y_h + ... + lambda_nu * X_h^j * Y_h = 0

Просуммируем по всем h = 1, ..., nu. По определению S_m = SUM_h Y_h * X_h^m, получаем:

    S_(j+nu) + lambda_1 * S_(j+nu-1) + lambda_2 * S_(j+nu-2) + ... + lambda_nu * S_j = 0

Это **линейное** уравнение в неизвестных lambda_1, ..., lambda_nu. Меняя j от 1 до nu, получаем nu таких уравнений.

---

## 9. Запись системы для nu = 3

Для случая трёх ошибок (nu = 3) система уравнений (j = 1, 2, 3):

    S_4 + lambda_1 * S_3 + lambda_2 * S_2 + lambda_3 * S_1 = 0
    S_5 + lambda_1 * S_4 + lambda_2 * S_3 + lambda_3 * S_2 = 0
    S_6 + lambda_1 * S_5 + lambda_2 * S_4 + lambda_3 * S_3 = 0

В матричной форме (свободные члены в правую часть):

    | S_1  S_2  S_3 |   | lambda_3 |     | -S_4 |
    | S_2  S_3  S_4 | * | lambda_2 |  =  | -S_5 |
    | S_3  S_4  S_5 |   | lambda_1 |     | -S_6 |

Все S_j известны (вычислены подстановкой alpha^j в принятый полином v(x)). Система линейная относительно lambda_1, lambda_2, lambda_3 -- решается стандартными методами.

---

## 10. Полный алгоритм Питерсона-Горенстейна-Цирлера

1. **Вычисление синдромов.** По принятому слову v(x) считаем S_j = v(alpha^j) для j = 1, ..., d - 1.

2. **Решение линейной системы.** Из системы уравнений (вида, показанного выше для nu = 3) находим коэффициенты lambda_1, ..., lambda_nu полинома локатора ошибок.

3. **Поиск корней Lambda(x).** Перебором по всем элементам поля (как при декодировании БЧХ на прошлой лекции) находим корни. Каждый найденный корень есть X_h^(-1).

4. **Локализация ошибок.** Берём обратные величины: X_h = (X_h^(-1))^(-1). Так как X_h = alpha^(i_h), степень alpha указывает позицию ошибки i_h.

5. **Нахождение значений ошибок.** Теперь X_h известны. Подставляем их в исходную систему:

       S_j = Y_1 * X_1^j + Y_2 * X_2^j + ... + Y_nu * X_nu^j

   Это **линейная** система относительно Y_1, ..., Y_nu (так как X_h известны, X_h^j -- известные коэффициенты). Решаем и находим значения ошибок.

6. **Исправление.** Зная позиции i_h и значения Y_h, строим вектор ошибки e(x) и получаем кодовое слово:

       c(x) = v(x) - e(x)

Замечание о нелинейности. На первом шаге мы ушли от нелинейности в X через переход к полиному локатора ошибок Lambda(x). Когда X_h найдены, оставшаяся задача нахождения Y_h уже линейная.

## 11. Пример построения РС-кода для d = 7

Возьмём то же поле GF(2^4) с примитивным полиномом 1 + x + x^4.

Хотим d = 7, тогда:

    k = n - d + 1 = 15 - 7 + 1 = 9

Код (15, 9), исправляющий t = floor((7-1)/2) = 3 ошибки.

### Порождающий полином

При b = 1, d - 1 = 6 множителей:

    g(x) = (x + alpha)(x + alpha^2)(x + alpha^3)(x + alpha^4)(x + alpha^5)(x + alpha^6)

(Знаки + и - совпадают в характеристике 2.)

### Частичное вычисление коэффициентов

После раскрытия g(x) = x^6 + (...) * x^5 + ...

**Коэффициент при x^5.** По формулам Виета это сумма всех корней (со знаком, который в GF(2) роли не играет):

    alpha + alpha^2 + alpha^3 + alpha^4 + alpha^5 + alpha^6

Используем разложения из таблицы поля GF(2^4):

    alpha^4 = 1 + alpha
    alpha^5 = alpha + alpha^2
    alpha^6 = alpha^2 + alpha^3

Подставляем:

    alpha + alpha^2 + alpha^3 + (1 + alpha) + (alpha + alpha^2) + (alpha^2 + alpha^3)

Группируем по степеням:

    1: 1
    alpha: alpha + alpha + alpha = alpha       (1+1+1 = 1 в GF(2))
    alpha^2: alpha^2 + alpha^2 + alpha^2 = alpha^2
    alpha^3: alpha^3 + alpha^3 = 0

Итого: 1 + alpha + alpha^2 = **alpha^10** (по таблице).

То есть:

    g(x) = x^6 + alpha^10 * x^5 + (...) * x^4 + ...

Полное вычисление остальных коэффициентов оставлено в качестве домашнего задания.

## 12. Замечания о реализации алгоритма

### Известность числа ошибок

В формальном описании алгоритма Питерсона-Горенстейна-Цирлера число ошибок nu заранее неизвестно. Полная реализация перебирает значения nu от t вниз и проверяет согласованность системы.

В рамках данного курса мы упрощаем: считаем nu заранее известным (например, заданным условием задачи).

### Сводка идеи алгоритма

Получили принятое слово -> вычислили синдромы -> нашли коэффициенты полинома локатора ошибок (линейная система) -> нашли корни (перебором) -> локализовали ошибки (взятие обратных) -> нашли значения ошибок (вторая линейная система) -> вычли ошибку из принятого слова.

В первый раз решаем линейную систему для нахождения позиций ошибок (через полином локатора, чтобы уйти от нелинейности). Во второй раз решаем линейную систему уже для значений ошибок -- это становится возможным после того, как X_h стали известными.
