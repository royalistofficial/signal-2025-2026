# Конспект лекции 4: Декодирование по максимуму правдоподобия и решётчатое представление кодов

## 1. Дополнение: классы эквивалентности (смежные классы)

### Повторение

Есть код C, состоящий из кодовых слов c1, c2, ..., cM (всего 2^k слов из пространства 2^n векторов).

Если к каждому кодовому слову прибавить элемент a:

    {c1 + a, c2 + a, ..., cM + a}

получим новое множество из 2^k элементов.

### Когда множества совпадают

Если a -- кодовое слово, то {c + a} = C (тот же код, перенумерованный). Это следствие линейности: сумма двух кодовых слов -- кодовое слово.

Таких элементов a (кодовых слов) ровно 2^k.

### Когда множества различны

Если a -- не кодовое слово, то {c + a} -- совершенно другое множество, не пересекающееся с C.

Пусть теперь b -- другой вектор (b != a, b не из C). Когда {c + a} = {c + b}?

Это происходит, когда существуют ci, cj такие, что ci + a = cj + b. Тогда a + b = ci + cj -- кодовое слово. То есть множества совпадают тогда и только тогда, когда a + b принадлежит коду.

Для каждого фиксированного a существует ровно 2^k таких b (по числу кодовых слов).

### Итого

Всё пространство 2^n векторов разбивается на:

    2^n / 2^k = 2^(n-k) = 2^r

непересекающихся смежных классов (классов эквивалентности), в каждом по 2^k элементов.

## 2. Дискретный канал и функция правдоподобия

### Дискретный канал без памяти

Канал задаётся множеством входных символов X, выходных символов Y и переходными вероятностями P(y | x) -- вероятность получить y при передаче x.

Канал без памяти: элементы передаются независимо, поэтому:

    P(y | c) = PROD(i=1..n) P(y_i | c_i)

### Функция правдоподобия

Передаём кодовые слова из кода C = {c1, c2, ..., cM}. Из канала пришла последовательность y.

Функция правдоподобия для m-го кодового слова:

    P(y | c_m) = PROD(i=1..n) P(y_i | c_m,i)

### Декодирование по максимуму правдоподобия (ML)

Выбираем кодовое слово, максимизирующее функцию правдоподобия:

    c* = argmax_m P(y | c_m)

### Логарифм функции правдоподобия

Для удобства переходим к логарифму (произведение -> сумма):

    Lambda_m(y) = log P(y | c_m) = SUM(i=1..n) log P(y_i | c_m,i)

Обозначение: lambda(y_i, c_m,i) = log P(y_i | c_m,i).

Тогда:

    Lambda_m(y) = SUM(i=1..n) lambda(y_i, c_m,i)

Задача: найти m, при котором Lambda_m(y) максимально.

## 3. Отношение правдоподобия и вектор надёжности

### Отношение правдоподобия (для двоичного случая)

Для каждого принятого символа y определим:

    L(y) = log [ P(y | 1) / P(y | 0) ]

Свойства:

- Если реально передана 1: P(y=1|1) > P(y=1|0), значит L(y) > 0.
- Если реально передан 0: P(y=0|0) > P(y=0|1), значит L(y) < 0.
- Знак L(y) указывает наиболее вероятный переданный символ.

### Вектор надёжности

Для принятого вектора y = (y1, y2, ..., yn) составляем вектор надёжности:

    L(y) = (L(y1), L(y2), ..., L(yn))

### Преобразование кодового слова

Для m-го кодового слова c_m = (c_m,1, ..., c_m,n) определим:

    xi_m,i = 2 * c_m,i - 1

То есть:

    c_m,i = 0  =>  xi_m,i = -1
    c_m,i = 1  =>  xi_m,i = +1

Вектор xi_m = (xi_m,1, ..., xi_m,n).

## 4. Теорема: ML-декодирование через скалярное произведение

### Формулировка

Для двоично-симметричного канала декодирование по максимуму правдоподобия сводится к поиску кодового слова c_m, для которого максимально скалярное произведение:

    <xi_m, L(y)> = SUM(i=1..n) xi_m,i * L(y_i)

### Доказательство (схема)

Рассмотрим два кодовых слова c_m и c_m'. Разобьём все n позиций на четыре множества:

    I(0,0) = {i : c_m,i = 0 и c_m',i = 0}
    I(1,1) = {i : c_m,i = 1 и c_m',i = 1}
    I(1,0) = {i : c_m,i = 1 и c_m',i = 0}
    I(0,1) = {i : c_m,i = 0 и c_m',i = 1}

Разность скалярных произведений:

    <xi_m, L(y)> - <xi_m', L(y)>

На множествах I(0,0) и I(1,1) слагаемые одинаковы и сокращаются. Остаются только I(1,0) и I(0,1):

    = 2 * SUM(i in I(1,0)) L(y_i) - 2 * SUM(i in I(0,1)) L(y_i)

После подстановки определения L(y_i) и алгебраических преобразований получаем:

    <xi_m, L(y)> - <xi_m', L(y)> = 2 * [Lambda_m(y) - Lambda_m'(y)]

То есть разности скалярных произведений пропорциональны разностям логарифмов правдоподобия. Следовательно, максимизация скалярного произведения эквивалентна максимизации функции правдоподобия.   QED

## 5. ML-декодирование = декодирование по минимуму расстояния Хэмминга (в ДСК)

### Отношение правдоподобия в ДСК

Двоично-симметричный канал с вероятностью ошибки p0 < 1/2:

    L(y) = log((1-p0)/p0)   если y = 1
    L(y) = -log((1-p0)/p0)  если y = 0

Обозначим delta = log((1-p0)/p0) > 0 (т.к. p0 < 1/2).

### Скалярное произведение через расстояние Хэмминга

Подставляя в скалярное произведение:

    <xi_m, L(y)> = delta * (n - 2 * d(y, c_m))

где d(y, c_m) -- расстояние Хэмминга между принятым словом и кодовым.

### Вывод

При delta > 0 (т.е. p0 < 1/2):

    <xi_m, L(y)> -> max   <=>   d(y, c_m) -> min

Таким образом, в двоично-симметричном канале декодирование по максимуму правдоподобия эквивалентно декодированию по минимуму расстояния Хэмминга.

Расстояние Хэмминга -- не просто абстрактная метрика, а оптимальный критерий декодирования в ДСК.

## 6. Решётчатое представление кодов

### Определение решётки

Решётка (trellis) -- направленный граф со специальной структурой:

1. Вершины разбиты на непересекающиеся множества -- ярусы (уровни), пронумерованные от 0 до N.
2. Нулевой и последний ярусы содержат ровно одну вершину.
3. Рёбра направлены только от яруса с меньшим номером к ярусу с большим (соседнему).
4. Рёбрам приписаны метки (кодовые символы) и веса.

Длина пути = сумма весов рёбер, входящих в путь.

### Профиль решётки

Профиль -- вектор, показывающий число узлов на каждом ярусе. Определяет сложность декодера: чем меньше узлов, тем проще декодер.

### Пример: код (3, 2) с проверкой на чётность

Порождающая матрица:

|   | 1 | 2 | 3 |
|---|---|---|---|
| g1| 1 | 1 | 0 |
| g2| 0 | 1 | 1 |

Кодовые слова: {000, 011, 101, 110}.

Решётка (4 яруса: 0, 1, 2, 3):

    Ярус 0: одна вершина (начало)
    Ярус 1: две вершины (символ 0 или 1)
    Ярус 2: две вершины (после схлопывания)
    Ярус 3: одна вершина (конец)

Профиль: (1, 2, 2, 1).

Рёбра помечены кодовыми символами. Каждый путь от начала до конца соответствует кодовому слову.

Ключевое наблюдение: на ярусе 2 происходит схлопывание -- пути с общим будущим объединяются в один узел. Это уменьшает число вершин и сложность декодера.

## 7. Декодирование по решётке (пример)

### Постановка задачи

Код (3, 2) с кодовыми словами {000, 011, 101, 110}. Из канала принято y = (1, 0, 0).

### Пошаговый алгоритм

Ярус 0 -> 1: первый символ y1 = 1.

    Ребро с меткой 0: d(1, 0) = 1. Накопленный вес = 1.
    Ребро с меткой 1: d(1, 1) = 0. Накопленный вес = 0.

Ярус 1 -> 2: второй символ y2 = 0.

    Из узла с весом 1 (символ был 0):
        Ребро 0: d(0,0)=0, вес = 1+0 = 1
        Ребро 1: d(0,1)=1, вес = 1+1 = 2  (отбрасываем -- худший путь)
    Из узла с весом 0 (символ был 1):
        Ребро 0: d(0,0)=0, вес = 0+0 = 0
        Ребро 1: d(0,1)=1, вес = 0+1 = 1

В узлах яруса 2 сравниваем входящие пути и оставляем лучшие (с меньшим весом).

Ярус 2 -> 3: третий символ y3 = 0.

    Вычисляем аналогично и находим пути с минимальным суммарным расстоянием Хэмминга.

### Результат

Три кодовых слова {000, 101, 110} имеют расстояние 1 от y = (1,0,0). Слово 011 отброшено на раннем этапе (расстояние 2).

Декодер может выбрать любое из трёх слов с расстоянием 1 (неоднозначность при d_min = 2).

## 8. Построение решётки для произвольного кода

### Общая процедура

Для кода C длины n решётка имеет (n + 1) ярусов (от 0 до n).

Каждое кодовое слово c_m = (c_m,1, ..., c_m,n) разбивается в точке i на две части:

    c_m = (c_m^P, c_m^F)

где c_m^P -- прошлое (past, первые i символов), c_m^F -- будущее (future, оставшиеся n-i символов).

### Множество будущих путей

Для данного прошлого c^P на ярусе i определим:

    F_i(c^P) = {c^F : (c^P, c^F) принадлежит C}

-- все возможные продолжения (будущие) для данного прошлого.

### Правило объединения узлов

Два узла на одном ярусе можно объединить (схлопнуть), если у них совпадают множества будущих путей:

    F_i(c1^P) = F_i(c2^P)  =>  узлы объединяются

Это ключ к минимизации решётки.

### Правило соединения рёбрами

Вершина V на ярусе i соединяется ребром с вершиной V' на ярусе i+1, если существует кодовое слово, прошлое которого (длины i+1) заканчивается символом, помечающим это ребро, а прошлое длины i совпадает с прошлым вершины V.

### Таблица построения

Для каждого яруса заполняется таблица:

| Столбец | Содержание |
|---------|------------|
| Номер яруса | i |
| c^P | Все варианты прошлого (первых i символов) |
| F_i(c^P) | Множество будущих путей для каждого c^P |
| Предыдущие вершины | Откуда пришли пути |
| Вершина | Номер узла на текущем ярусе |
| Кодовый символ | Метка ребра (i-й символ кодового слова) |

### Пример: код (6, 3)

Код из 8 кодовых слов длины 6. Решётка имеет 7 ярусов (от 0 до 6).

Ярус 0: одна вершина, прошлое пусто, будущее = все 8 кодовых слов.

Ярус 1: два варианта прошлого (0 или 1). Для каждого -- свои 4 будущих слова. Два узла.

Ярус 2: четыре варианта прошлого (00, 01, 10, 11). Каждому соответствует по 2 будущих слова. Четыре узла.

Далее -- аналогично, с возможным схлопыванием узлов на старших ярусах (когда будущее совпадает).

## 9. Сложность декодирования по решётке

Сложность определяется профилем решётки -- числом узлов на каждом ярусе.

Для одного и того же кода можно построить различные решётки (зависит от порядка координат, способа схлопывания). Задача -- найти решётку с минимальным профилем.

Преимущество решётки: на каждом ярусе, где сливаются пути, можно отбросить заведомо проигрышные (с большим накопленным расстоянием). Это аналог алгоритма Витерби.

## 10. Итоги лекции

Ключевые результаты:

1. В ДСК декодирование по максимуму правдоподобия эквивалентно декодированию по минимуму расстояния Хэмминга (при p < 1/2).

2. Любой блоковый код можно представить в виде решётки, где каждый путь соответствует кодовому слову.

3. Декодирование по решётке позволяет отбрасывать неоптимальные пути на каждом ярусе, что снижает сложность по сравнению с полным перебором.

4. Сложность решётки определяется её профилем (числом узлов на ярусах). Для линейных кодов решётки компактнее за счёт схлопывания узлов с одинаковым будущим.
