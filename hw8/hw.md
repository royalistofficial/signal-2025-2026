## Задача 1. Порядки всех элементов мультипликативной группы GF(5^2)

### Условие

Для мультипликативной группы поля GF(5^2), считая X примитивным элементом, определить порядки всех элементов поля.

### Постановка

Мультипликативная группа поля GF(5^2) содержит q - 1 = 24 элемента. Так как X -- примитивный элемент, все элементы группы имеют вид X^k, k = 1, 2, ..., 24 (причём X^24 = 1).

Из лекции известно, что порядок любого элемента делит 24. Делители 24:

    1, 2, 3, 4, 6, 8, 12, 24

Задача -- найти порядок каждого X^k.

### Формула порядка

**Утверждение.** Для примитивного элемента X порядка n = 24 порядок произвольного X^k вычисляется по формуле:

    ord(X^k) = n / gcd(n, k)

### Доказательство

Обозначим ord(X^k) = d. По определению d -- наименьшее положительное число, такое что (X^k)^d = 1, то есть X^(k*d) = 1.

Так как X имеет порядок n, равенство X^(k*d) = 1 эквивалентно тому, что n | k*d.

Пусть g = gcd(n, k). Запишем n = g * n', k = g * k', где gcd(n', k') = 1.

Условие n | k*d принимает вид g*n' | g*k'*d, то есть n' | k'*d. Так как gcd(n', k') = 1, получаем n' | d.

Минимальное такое d равно n' = n / g = n / gcd(n, k).   QED

### Вычисление порядков

Для n = 24 перебираем k = 1, ..., 24:

| 9  | 3          | 8                    |
| 10 | 2          | 12                   |
| 11 | 1          | 24                   |
| 12 | 12         | 2                    |
| 13 | 1          | 24                   |
| 14 | 2          | 12                   |
| 15 | 3          | 8                    |
| 16 | 8          | 3                    |
| 17 | 1          | 24                   |
| 18 | 6          | 4                    |
| 19 | 1          | 24                   |
| 20 | 4          | 6                    |
| 21 | 3          | 8                    |
| 22 | 2          | 12                   |

| 24 | 24         | 1                    |

### Проверка через функцию Эйлера

Для циклической группы порядка n число элементов порядка d (где d | n) равно phi(d) (функция Эйлера):

    phi(1) = 1
    phi(2) = 1
    phi(3) = 2
    phi(4) = 2
    phi(6) = 2
    phi(8) = 4
    phi(12) = 4
    phi(24) = 8

Сумма: 1 + 1 + 2 + 2 + 2 + 4 + 4 + 8 = 24 = n.   ok

Значения из таблицы совпадают с phi(d) -- распределение верное.

### Примитивные элементы

Примитивный элемент -- элемент максимального порядка 24. Их всего phi(24) = 8 штук:

    X, X^5, X^7, X^11, X^13, X^17, X^19, X^23

Индексы k этих элементов -- это в точности числа, взаимно простые с 24.

Именно они порождают всю мультипликативную группу своими степенями. В качестве примитивного элемента можно взять любой из них, полная группа будет построена.

### Вывод

Все порядки являются делителями 24, что подтверждает теорему из лекции. Количество элементов каждого порядка d задаётся функцией Эйлера phi(d). Примитивных (образующих) элементов ровно 8.

## Задача 2. Кодер (15, 4)-кода с h(x) = 1 + x + x^4

### Условие

Для кодера (15, 4)-кода с проверочным полиномом h(x) = 1 + x + x^4 описать смену состояний регистра при начальном заполнении (0, 0, 1, 1). Убедиться, что это код максимальной длины. Определить минимальное расстояние.

### Полученный код

Проверочный полином h(x) = 1 + x + x^4 -- это примитивный полином степени 4 над GF(2). Так как он берётся именно как проверочный, получается код, дуальный коду Хэмминга -- симплексный код, он же код максимальной длины.

Параметры:

    n = 2^4 - 1 = 15
    k = deg h(x) = 4
    код (15, 4)

### Рекуррентное соотношение

Для циклического кода с проверочным полиномом h(x) кодовое слово c(x) удовлетворяет c(x) * h(x) ≡ 0 mod (x^n - 1). В коэффициентах это даёт линейную рекурренту:

    h_0 * c_i + h_1 * c_(i+1) + ... + h_m * c_(i+m) = 0

Для h(x) = 1 + x + x^4 получаем h_0 = 1, h_1 = 1, h_4 = 1:

    c_i + c_(i+1) + c_(i+4) = 0

Откуда:

    c_(i+4) = c_i + c_(i+1)       (в GF(2): + это XOR)

Это рекуррента, которую реализует сдвиговый регистр с обратной связью.

### Схема кодера

Четыре ячейки M_0, M_1, M_2, M_3. На каждом такте:

1. Выход -- содержимое M_0 (старший разряд, выходит наружу как бит кодового слова).
2. Содержимое всех ячеек сдвигается: M_0 -> (выход), старое M_1 -> новое M_0, старое M_2 -> новое M_1, старое M_3 -> новое M_2.
3. В освободившуюся ячейку M_3 записывается M_0 XOR M_1 (из старых значений) -- обратная связь, соответствующая h_0 и h_1.

Правило смены состояния:

    (M_0, M_1, M_2, M_3)  ->  (M_1, M_2, M_3, M_0 XOR M_1)
    выход = M_0

### Смена состояний при начальном заполнении (0, 0, 1, 1)

| Такт | M_0 | M_1 | M_2 | M_3 | Выход (бит КС) |
|------|-----|-----|-----|-----|----------------|
| 0    | 0   | 0   | 1   | 1   | --             |
| 1    | 0   | 1   | 1   | 0   | 0              |
| 2    | 1   | 1   | 0   | 1   | 0              |
| 3    | 1   | 0   | 1   | 0   | 1              |
| 4    | 0   | 1   | 0   | 1   | 1              |
| 5    | 1   | 0   | 1   | 1   | 0              |
| 6    | 0   | 1   | 1   | 1   | 1              |
| 7    | 1   | 1   | 1   | 1   | 0              |
| 8    | 1   | 1   | 1   | 0   | 1              |
| 9    | 1   | 1   | 0   | 0   | 1              |
| 10   | 1   | 0   | 0   | 0   | 1              |
| 11   | 0   | 0   | 0   | 1   | 1              |
| 12   | 0   | 0   | 1   | 0   | 0              |
| 13   | 0   | 1   | 0   | 0   | 0              |
| 14   | 1   | 0   | 0   | 1   | 0              |
| 15   | 0   | 0   | 1   | 1   | 1              |

После такта 15 состояние регистра возвращается к исходному (0, 0, 1, 1) -- цикл замкнулся.

Примеры вычислений первых тактов:

    Такт 1: было (0,0,1,1). Выход = M_0 = 0.
            M_0 XOR M_1 = 0 XOR 0 = 0.
            Новое состояние: (0, 1, 1, 0).

    Такт 2: было (0,1,1,0). Выход = 0.
            0 XOR 1 = 1. Новое состояние: (1, 1, 0, 1).

    Такт 3: было (1,1,0,1). Выход = 1.
            1 XOR 1 = 0. Новое состояние: (1, 0, 1, 0).

### Подтверждение: код максимальной длины

Проверяем два свойства.

**1. Все 15 ненулевых состояний различны.**

Выпишем состояния (M_0, M_1, M_2, M_3) на тактах 0-14:

    0011, 0110, 1101, 1010, 0101, 1011, 0111, 1111,
    1110, 1100, 1000, 0001, 0010, 0100, 1001

Это все 15 ненулевых 4-битных векторов (отсутствует только 0000).

**2. Нулевое состояние недостижимо.**

Если (M_0, M_1, M_2, M_3) = (0, 0, 0, 0), то следующее состояние снова (0, 0, 0, 0), и обратно в любое ненулевое состояние не попасть. Значит, стартуя с ненулевого состояния, нулевого не достигнем.

**3. Период равен 15.**

После 15 тактов регистр возвращается в исходное состояние -- значит, выходная последовательность имеет период 15 = 2^m - 1, что является теоретическим максимумом для 4-битного регистра. Это и есть определение кода максимальной длины (M-последовательность).

### Кодовое слово и его вес

Выходная последовательность (такты 1-15):

    c = (0, 0, 1, 1, 0, 1, 0, 1, 1, 1, 1, 0, 0, 0, 1)

Вес: w(c) = 0+0+1+1+0+1+0+1+1+1+1+0+0+0+1 = **8**.

### Проверка рекурренты

Вычислим c_(i+4) = c_i + c_(i+1) напрямую от (c_0, c_1, c_2, c_3) = (0, 0, 1, 1):

    c_14 = c_10 + c_11 = 1 + 0 = 1

Для c = (0,0,1,1,0,1,0,1,1,1,1,0,0,0,1) проверим соотношение c_i + c_(i+1) + c_(i+4) = 0 по модулю 15:

### Минимальное расстояние

Для кода, дуального коду Хэмминга (симплексного кода), было доказано в ДЗ 2 и ДЗ 5, что **все ненулевые кодовые слова имеют один и тот же вес** 2^(m-1).

Для m = 4:

    d_min = 2^(m-1) = 2^3 = 8

Вес полученного кодового слова w(c) = 8 это подтверждает.

### Итоговые выводы

1. Регистр проходит все 15 ненулевых состояний за 15 тактов и возвращается в исходное -- это **код максимальной длины** (M-последовательность, длина периода = 2^m - 1).
